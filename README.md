# RoyalFight

RoyalFight 是一个基于 Godot 4 的双路即时对战原型，玩法灵感来自《皇室战争》，目标是先把核心战斗循环、卡牌部署、双路推进和塔防交互做成一个可玩、可继续扩展的实验项目。

## 项目概述

当前版本重点放在这些体验上：
- 双路推进与防守。
- 圣水恢复和卡牌部署。
- 单位自动寻路、索敌与战斗。
- 公主塔与国王塔的攻防节奏。
- 一个足够清晰、方便继续扩展的代码结构。

这是一个可玩原型，不是内容完整的正式游戏版本。

## 当前已实现内容

核心玩法：
- 左右双路战场与河道桥梁结构。
- 4 手牌轮转与下一张预览。
- 4 张基础卡牌：Knight、Archer、Giant、Mini P.E.K.K.A。
- 玩家下半场部署与敌方自动出兵 AI。
- 圣水自动回复与费用消耗。
- 单位自动前进、自动索敌、自动攻击。
- 左右公主塔与国王塔。
- 国王塔唤醒逻辑。
- 目标记忆、仇恨切换与重新索敌。
- 近战卡位、友军阻挡、简单绕行。
- 攻击前摇、后摇、受击停顿与轻微击退。
- 远程单位与塔的弹道可视化和命中结算。
- 程序化攻击 / 命中音效。
- 使用 SVG / 贴图素材替换原始圆形占位图标。
- Android 竖屏与导出配置。

最近一轮额外修复：
- 修正弹道命中解算，避免箭飞出去但不掉血。
- 给单位加上桥口导航和河道碰撞约束，避免直接下河。
- 玩家部署位置按点击点稳定落位，不再随机偏到路两侧。
- 手机手牌文案缩短，避免 `Mini P.E.K.K.A` 在窄屏上撑坏布局。
- 修正受击 / 收招动画数值过冲，避免单位异常放大和闪烁。

胜负规则：
- 摧毁敌方国王塔即获胜。
- 己方国王塔被摧毁则失败。

## 操作方式

- 点击底部手牌选择卡牌。
- 在己方半场点击左路或右路进行部署。
- 手机触屏与桌面鼠标都可以直接部署。
- 单位部署后会自动移动和战斗。
- 对局结束后可以点击重开按钮重新开始。

## 项目结构

- `project.godot`：Godot 项目配置。
- `Main.tscn`：主场景入口。
- `scripts/main.gd`：战场流程、UI、出兵、目标选择、导航、音效和整局控制。
- `scripts/battle_unit.gd`：单位移动、攻击、索敌、受击和单位绘制逻辑。
- `scripts/battle_building.gd`：塔的攻击、受击和建筑绘制逻辑。
- `assets/units/*.svg`：单位图标素材。
- `assets/buildings/*.svg`：塔的图标素材。
- `.gitignore`：项目级忽略规则，避免把 `.godot/` 和导出产物提交进仓库。

## 美术与音频说明

美术：
- 单位和塔不再使用纯圆形 / 方块占位，而是接入本地素材。
- 底部手牌按钮与战场单位共用同一套图标资源，视觉上更统一。
- 运行时会做纹理缓存，减少重复加载。

音频：
- 当前攻击音效和命中音效为代码生成的程序化短音频。
- 项目目前不依赖外部音效文件。

## 运行方式

环境要求：
- Godot 4.6.x

运行步骤：
1. 用 Godot 打开当前项目目录。
2. 直接运行默认主场景，或手动运行 `Main.tscn`。

- 继续优化桥头会战密度和寻路质量。

## 当前限制

这个项目目前仍然是原型，所以还没有做这些内容：
- 完整的《皇室战争》卡牌系统。
- 联机或多人对战。
- 成熟的数值平衡。
- 成套角色动画和正式量产美术。
- 长线成长、匹配或商业化系统。

## 说明

本项目是一个受《皇室战争》玩法结构启发的原创练习型原型，仅用于实验、学习和快速迭代，不是 Supercell 官方项目。
